## Real fixes landed

1. **iter13 — root `$I30` indexes every system record.** Was the
   reason chkdsk emitted 12 "Detected orphaned file <name>" lines
   in Stage 2 and bailed at frs.cxx 60F. Confirmed on VM: orphan
   lines 12 → 0. Stage 1 + Stage 2 both verify cleanly.

2. **iter14 — `$SECURITY_DESCRIPTOR` (0x50) on every system record +
   compact 48-byte `$STANDARD_INFORMATION`.** Was the reason 7 of 12
   system records had attribute-set diffs from Microsoft's reference.
   Post-iter14, records 1, 2, 4, 6, 7, 8, 10 are byte-identical to
   reference. iter14 does NOT fully clear the trailing frs.cxx 60F at
   end of Stage 2; the residual must be in one of the still-divergent
   records (rec 0/3/5/9/11) or a global check we haven't isolated.

Linux test contract `cargo test --release --lib mkfs --test
mkfs_roundtrip --test mkfs_bin_smoke` = 7/7 passing throughout
(both fixes required updating `format_and_parse_back` to assert the
new — correct — root index population).
